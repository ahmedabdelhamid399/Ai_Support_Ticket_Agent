# Airtable Schema — Tickets Table

## Base Name: Support Ticket Agent
## Table Name: Tickets

| Field Name | Type | Description |
|---|---|---|
| Ticket ID | Autonumber | Auto-generated unique ID |
| Title | Single line text | Brief ticket subject |
| User | Single line text | Name of person who submitted |
| Customer Email | Email | Contact email address |
| Issue | Long text | Full description of the problem |
| Priority | Single select | Low / Medium / High / Critical |
| Date Submitted | Date | When ticket was created |
| Status | Single select | New / Processing / Summarized / Skipped / Error |
| AI Summary | Long text | AI generated summary |
| Email Subject | Single line text | AI generated email subject |
| Email Content | Long text | AI generated email body |
| Processed At | Date | When workflow processed this ticket |
| Error Log | Long text | Error details if workflow failed |

## Status Flow

\`\`\`
New → Processing → Summarized
New → Processing → Skipped (missing fields)
New → Processing → Error (workflow failed)
\`\`\`

## Notes
- Ticket ID is auto-generated by Airtable
- Priority is set automatically by Groq AI
- All AI fields are populated by n8n workflow
- Processed At timestamp is set by n8n automatically
